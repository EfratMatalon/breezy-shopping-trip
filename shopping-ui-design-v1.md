## 1. חזון המוצר

### 1.1 מטרות המוצר

אפליקציית רשימות קניות שיתופית למשק בית. המטרה: לאפשר לכל בני המשפחה לנהל רשימת קניות אחת, משותפת ומסונכרנת בזמן אמת, בפשטות מוחלטת ובמינימום חיכוך.

**מטרות מרכזיות:**

- **שיתוף מיידי** — כל בן משפחה רואה את אותה רשימה, מתעדכנת בזמן אמת. מי שבסופר רואה מה שמישהו אחר הוסיף לפני שנייה.
- **מהירות הוספה** — הוספת פריט חייבת להיות הפעולה הכי מהירה באפליקציה. שלוש הקשות מקסימום מכל מסך.
- **אפס עומס קוגניטיבי** — המשתמש לא צריך ללמוד את האפליקציה. הכל מובן בהצצה ראשונה.
- **המשכיות בין סשנים** — רשימה פעילה, היסטוריית קניות, וחברי משק בית נשמרים בין שימושים.
- **שליטה במשק הבית** — יצירה, הצטרפות, ניהול חברים ועזיבה — בלי בלבול.

### 1.2 קהל היעד

- **משפחות ובני זוג** המנהלים משק בית משותף ורוצים רשימת קניות אחת לכולם.
- **שותפים לדירה** המתחלקים בקניות הבית.
- **טווח גילאים רחב** — מ-18 עד 70+. ה-UI חייב להיות נגיש גם למשתמשים שאינם טכנולוגיים.
- **שימוש בעיקר במובייל**, תוך כדי תנועה ובתוך הסופר. שימוש משני בדסקטופ לתכנון.

### 1.3 מסעות משתמש מרכזיים (Core User Journeys)

**מסע א' — משתמש חדש יוצר משק בית:**
התחברות → Onboarding → יצירת משק בית → קבלת קוד הזמנה → שיתוף הקוד → דשבורד ריק → הוספת פריט ראשון.

**מסע ב' — משתמש מצטרף למשק בית קיים:**
קבלת קישור הזמנה → התחברות → מסך הצטרפות עם קוד מולא מראש → אישור → דשבורד עם הרשימה המשותפת.

**מסע ג' — קנייה יומיומית (הליבה):**
פתיחת אפליקציה → דשבורד → רשימה פעילה → סימון פריטים כ"נקנו" תוך כדי הליכה בסופר → סיום קנייה → ארכוב הרשימה להיסטוריה.

**מסע ד' — הוספה מהירה מהבית:**
פתיחת אפליקציה → לחיצה על "+" → הקלדת פריט → שמירה → חזרה לרשימה. כל בני הבית רואים מיד.

**מסע ה' — ניהול משק הבית:**
הגדרות משק בית → צפייה בחברים → שיתוף קוד מחדש / הסרת חבר (יוצר) / עזיבת משק בית (חבר).

## 2. עקרונות עיצוב

הזהות הוויזואלית מבוססת על תמהיל: **70% Bring! / AnyList** (חמימות, צבעוניות פונקציונלית, כרטיסיות מוצר, ידידותיות) ו-**30% Apple** (היררכיה נקייה, מרווח נדיב, טיפוגרפיה מדויקת, מחוות ואנימציות עדינות).

### 2.1 פשטות (Simplicity)

- מסך אחד = מטרה אחת. אין עומס פעולות.
- הפעולה הראשית בכל מסך בולטת ויחידה (כפתור Primary אחד).
- ברירות מחדל חכמות מפחיתות החלטות מהמשתמש.
- טקסט קצר, פעלים ברורים, ללא ז'רגון.
- היררכיה ויזואלית מובילה את העין: כותרת → תוכן → פעולה.

### 2.2 נגישות (Accessibility)

- ניגודיות מינימלית AA (4.5:1 לטקסט רגיל, 3:1 לטקסט גדול).
- אזורי מגע מינימום 44×44px.
- כל פעולה נגישה גם דרך מקלדת.
- מצבי focus ברורים בכל אלמנט אינטראקטיבי.
- תמיכה ב-Dynamic Type / הגדלת גופן מערכת.
- צבע לעולם אינו האינדיקטור היחיד (תמיד מלווה באייקון/טקסט).

### 2.3 Mobile-First

- העיצוב מתחיל מ-375px ומתרחב כלפי מעלה.
- ניווט תחתון (bottom tab bar) במובייל — בהישג אגודל.
- הפעולה הראשית (הוספת פריט) צפה ונגישה תמיד (FAB).
- אינטראקציות מבוססות מגע: swipe לסימון/מחיקה, pull-to-refresh.
- RTL מלא — האפליקציה בעברית, כל הפריסה מימין לשמאל.

### 2.4 שיתוף פעולה משפחתי (Family Collaboration)

- סנכרון בזמן אמת — שינוי של אחד מופיע מיד אצל כולם.
- שקיפות פעולה — רואים מי הוסיף/סימן פריט (אווטר קטן או שם).
- אין "בעלות" על פריט — כולם שווים בעריכת הרשימה (פרט לפעולות יוצר ברמת משק הבית).
- מצב ריק מזמין שיתוף ("הזמינו את המשפחה").

### 3.5 הצללה / Elevation

מערכת הצללה עדינה ברוח Apple — צללים רכים, נמוכי opacity, ללא הגזמה.

| רמה | טוקן | ערך | שימוש |
|---|---|---|---|
| 0 | `shadow-none` | none | אלמנטים שטוחים על רקע |
| 1 | `shadow-card` | `0 1px 3px rgba(28,26,23,0.06), 0 1px 2px rgba(28,26,23,0.04)` | כרטיסים, פריטי רשימה |

## 4. אסטרטגיית Responsive

שלוש נקודות שבירה עיקריות. Mobile-first — ה-CSS הבסיסי הוא מובייל, ושאר ה-breakpoints מרחיבים.

| Breakpoint | טווח | טיפוס מכשיר |
|---|---|---|
| Mobile | 0–767px | טלפונים |
| Tablet | 768–1023px | טאבלטים, מסך מפוצל |
| Desktop | 1024px+ | מחשבים |

### 4.1 פריסת Mobile (0–767px)

- עמודה אחת, רוחב מלא עם gutter של `space-4` (16px).
- **ניווט תחתון** (Bottom Tab Bar) קבוע: רשימה / היסטוריה / הגדרות.
- **FAB** "הוסף פריט" צף בפינה התחתונה (מעל ה-tab bar).
- מודלים מופיעים כ-**Bottom Sheet** (עולים מלמטה, רדיוס עליון 24px).
- כותרת מסך עליונה (sticky) עם כותרת ופעולת back/menu.
- אינטראקציות swipe על פריטים.

### 4.2 פריסת Tablet (768–1023px)

- פריסה דו-עמודתית במסכים מתאימים: ניווט/רשימת רשימות בצד, תוכן ראשי.
- בדשבורד: סרגל צד צר עם משק הבית והניווט, אזור תוכן רחב לרשימה.
- מודלים מופיעים כ-**Centered Dialog** (לא bottom sheet), רוחב מקסימלי 480px.
- ה-FAB עובר לכפתור Primary בכותרת או נשאר צף לפי הקשר.
- gutter גדל ל-`space-6` (24px).

### 4.3 פריסת Desktop (1024px+)

- **שלושה אזורים**: סרגל ניווט קבוע ימני (256px), עמודת רשימה ראשית (גמישה, מקס' ~720px ממורכזת), אזור פרטים/היסטוריה אופציונלי.
- רוחב תוכן מוגבל (max-width) למניעת שורות ארוכות מדי — קריאות מעל הכל.
- ניווט תחתון מוחלף בניווט צדדי קבוע.
- hover states עשירים יותר (הפעולות מתגלות ב-hover על פריט).
- מודלים תמיד ממורכזים עם overlay מעומעם.
- קיצורי מקלדת זמינים (ראו פרק נגישות).

## 5. ארכיטקטורת ניווט

### 5.1 מפת המסלולים (Routes)

| מסלול | מסך | הגנה | תיאור |
|---|---|---|---|
| `/login` | Login | ציבורי | כניסה / הרשמה |
| `/auth/callback` | Auth Callback | ציבורי | טיפול בחזרה מ-OAuth / Magic Link |
| `/onboarding` | Onboarding | מאומת | קליטת משתמש חדש, בחירת מסלול |
| `/onboarding/create` | Create Household | מאומת | יצירת משק בית חדש |
| `/join` | Join Household | מאומת | הזנת קוד הזמנה ידנית |
| `/join/:code` | Join Household (prefilled) | מאומת | הצטרפות עם קוד מקישור |
| `/` | Shopping Dashboard | מאומת + משק בית | מסך הבית — רשימה פעילה |
| `/workspace` | Shopping List | מאומת + משק בית | תצוגת רשימת הקניות המלאה |

| `/settings/household` | Household Settings | מאומת + משק בית | ניהול חברים, קוד, עזיבה |

## 6. מפרטי מסכים

לכל מסך: מטרה, פעולות משתמש, קומפוננטות, מצבים (states), מצב ריק, מצב טעינה, מצב שגיאה.

### 6.1 Login (`/login`)

- **מטרה:** לאפשר כניסה/הרשמה במינימום חיכוך.
- **פעולות משתמש:** התחברות עם Google / Apple; כניסה עם Magic Link (אימייל); קריאת תנאי שימוש.
- **קומפוננטות:** לוגו + שם מוצר, כותרת ברוכים הבאים, כפתורי OAuth (Google/Apple), שדה אימייל + כפתור "שלח קישור", קישור תנאים/פרטיות, מפריד "או".
- **מצבים:** ברירת מחדל; אימייל הוקלד (כפתור פעיל); נשלח (אישור "בדקו את המייל"); כפתור OAuth ב-loading.
- **מצב ריק:** לא רלוונטי.
- **מצב טעינה:** ספינר על כפתור ה-OAuth/שליחה בזמן ההפניה.
- **מצב שגיאה:** באנר שגיאה מתחת לטופס ("ההתחברות נכשלה, נסו שוב"), שדה אימייל לא תקין מודגש באדום עם הודעת עזר.

### 6.2 Auth Callback (`/auth/callback`)

- **מטרה:** מסך מעבר בזמן עיבוד ההתחברות וההפניה ליעד הנכון.
- **פעולות משתמש:** אין — מסך פסיבי.
- **קומפוננטות:** לוגו ממורכז, ספינר, טקסט "מתחברים...".
- **מצבים:** טעינה (ברירת מחדל); הצלחה (הפניה אוטומטית); כישלון.
- **מצב ריק:** לא רלוונטי.
- **מצב טעינה:** זהו עצם המסך — אנימציית טעינה עדינה.
- **מצב שגיאה:** "אירעה שגיאה בהתחברות" + כפתור "חזרה להתחברות" → `/login`.

### 6.3 Onboarding (`/onboarding`)

- **מטרה:** לקלוט משתמש מאומת ללא משק בית ולנתב אותו ליצירה או הצטרפות.
- **פעולות משתמש:** "צרו משק בית חדש" → `/onboarding/create`; "הצטרפו למשק בית קיים" → `/join`; (אופציונלי) הזנת שם תצוגה.
- **קומפוננטות:** איור/אילוסטרציה ברוכים הבאים (placeholder), כותרת Display, שני כרטיסי בחירה גדולים (Create / Join), הסבר קצר לכל מסלול.
- **מצבים:** ברירת מחדל; כרטיס נבחר (highlight).
- **מצב ריק:** לא רלוונטי (זהו מצב התחלתי).
- **מצב טעינה:** skeleton קצר בזמן בדיקת סטטוס משק בית.
- **מצב שגיאה:** אם בדיקת המצב נכשלה — toast "טעינה נכשלה" + retry.

### 6.4 Create Household (`/onboarding/create`)

- **מטרה:** יצירת משק בית חדש בשלב אחד.
- **פעולות משתמש:** הזנת שם משק בית; (אופציונלי) בחירת אייקון/אמוג'י; "צרו"; back.
- **קומפוננטות:** שדה טקסט "שם משק הבית", בורר אייקון/צבע (אופציונלי), כפתור Primary "צרו משק בית", כפתור back.
- **מצבים:** ריק (כפתור disabled); תקין (כפתור פעיל); יצירה (loading); הצלחה (מעבר למסך קוד הזמנה / דשבורד).
- **מצב ריק:** placeholder בשדה ("לדוגמה: משפחת כהן").
- **מצב טעינה:** ספינר על כפתור "צרו" בזמן הכתיבה ל-DB.
- **מצב שגיאה:** "יצירת משק הבית נכשלה" באנר; שם ריק/ארוך מדי → ולידציה inline.

### 6.5 Join Household (`/join`, `/join/:code`)

- **מטרה:** הצטרפות למשק בית קיים באמצעות קוד.
- **פעולות משתמש:** הזנת קוד הזמנה (או מולא מראש מ-`:code`); "הצטרפו"; back.
- **קומפוננטות:** שדה קוד (input segmented, monospace), כפתור Primary "הצטרפו", הצגת שם משק הבית שזוהה (אם הקוד תקין), כפתור back.
- **מצבים:** ריק; קוד הוקלד; קוד מאומת (מציג שם משק בית + כפתור פעיל); הצטרפות (loading); הצלחה (→ דשבורד).
- **מצב ריק:** placeholder "הזינו קוד בן 6 תווים".
- **מצב טעינה:** בדיקת תקינות הקוד (ספינר קטן ליד השדה); הצטרפות (ספינר על הכפתור).
- **מצב שגיאה:** "קוד לא תקין או שפג תוקפו" מתחת לשדה (אדום); "כבר חבר במשק בית זה" → הפניה לדשבורד.

### 6.6 Household Settings (`/settings/household`)

- **מטרה:** ניהול משק הבית — חברים, קוד הזמנה, פעולות יוצר ועזיבה.
- **פעולות משתמש:** העתקת/שיתוף קוד הזמנה; צפייה ברשימת חברים; הסרת חבר (יוצר בלבד); עריכת שם משק הבית (יוצר); עזיבת משק בית; מחיקת משק בית (יוצר).
- **קומפוננטות:** כותרת מסך, כרטיס "קוד הזמנה" עם כפתורי העתק/שתף, רשימת חברים (Member Cards עם אווטר/שם/תפקיד), כפתור "הזמינו חברים", אזור פעולות מסוכנות (עזיבה/מחיקה — באדום).
- **מצבים:** חבר רגיל (רואה קוד + רשימה + עזיבה); יוצר (רואה גם הסרת חבר, עריכה, מחיקה); חבר יחיד.
- **מצב ריק:** משק בית עם חבר אחד → "אתם לבד כאן עדיין. הזמינו את המשפחה".
- **מצב טעינה:** skeleton לרשימת החברים.
- **מצב שגיאה:** "טעינת חברי משק הבית נכשלה" + retry; כשל בהסרה → toast שגיאה.

### 6.7 Shopping Dashboard (`/`)

- **מטרה:** מסך הבית. תצוגה מהירה של הרשימה הפעילה ונקודת ההוספה המהירה.
- **פעולות משתמש:** צפייה בפריטים; סימון פריט כ"נקנה" (checkbox/swipe); הוספת פריט (FAB); מעבר לרשימה מלאה / היסטוריה / הגדרות; pull-to-refresh.
- **קומפוננטות:** כותרת עם שם משק הבית + אווטרים של חברים, סיכום קצר ("12 פריטים · 3 נקנו"), רשימת Product Cards מקובצת לפי קטגוריה, FAB "+", Bottom Tab Bar.
- **מצבים:** רשימה מלאה; חלק נקנו; הכל נקנו (מצב "סיום קנייה" עם CTA לארכוב).
- **מצב ריק:** איור + "הרשימה ריקה. הוסיפו את הפריט הראשון" + כפתור גדול "הוסיפו פריט".
- **מצב טעינה:** skeleton של 4–5 שורות פריט.
- **מצב שגיאה:** "טעינת הרשימה נכשלה" + retry; כשל סנכרון → באנר עדין "לא מחובר — שינויים יסונכרנו מאוחר יותר".

### 6.8 Shopping List (`/workspace`)

- **מטרה:** תצוגת עבודה מלאה של הרשימה — קיבוץ, מיון, עריכה, סימון.
- **פעולות משתמש:** הוספה/עריכה/מחיקה של פריט; סימון נקנה; שינוי כמות; קיבוץ/מיון; חיפוש/סינון; ארכוב הרשימה.
- **קומפוננטות:** סרגל חיפוש/סינון, מתגי מיון (קטגוריה / א'-ב' / סדר הוספה), רשימת פריטים מקובצת עם כותרות קבוצה, פעולות swipe, FAB, כפתור "סיימתי לקנות" (ארכוב).
- **מצבים:** עריכה רגילה; מצב חיפוש פעיל; פריטים מסומנים (מודגשים/מקופלים לתחתית); כל הפריטים נקנו.
- **מצב ריק:** זהה לדשבורד — הזמנה להוסיף פריט ראשון.
- **מצב טעינה:** skeleton list.
- **מצב שגיאה:** כשל בעריכת פריט → הפריט חוזר למצב הקודם + toast; כשל טעינה → retry.

### 6.9 Add Item Modal

- **מטרה:** הוספת פריט במהירות מקסימלית.
- **פעולות משתמש:** הקלדת שם פריט; (אופציונלי) כמות, יחידה, קטגוריה, הערה; בחירה מהצעות אוטומטיות (פריטים נפוצים/אחרונים); שמירה; שמירה+הוספה נוספת; סגירה.
- **קומפוננטות:** שדה שם פריט (autofocus), צ'יפים של הצעות מהירות, בורר כמות, בורר קטגוריה, שדה הערה, כפתור "הוסף", כפתור "הוסף ועוד".
- **מצבים:** ריק (כפתור disabled); שם הוקלד (פעיל); הצעות מוצגות; שמירה (loading קצר); נוסף (toast + השדה מתאפס אם "הוסף ועוד").
- **מצב ריק:** הצעות מהירות מבוססות היסטוריה ("חלב", "לחם", "ביצים"...).
- **מצב טעינה:** ספינר זעיר על כפתור ההוספה (אופטימיסטי — הפריט מופיע מיד ברשימה).
- **מצב שגיאה:** כשל שמירה → הפריט מסומן "ממתין" + retry אוטומטי; toast "ההוספה נכשלה".

### 6.10 History (`/history`)

- **מטרה:** צפייה ברשימות שהושלמו/ארכיון, ושחזור פריטים.
- **פעולות משתמש:** עיון ברשימות עבר לפי תאריך; פתיחת רשימה שהושלמה; "הוסף שוב" לפריט/רשימה; חיפוש פריט בהיסטוריה.
- **קומפוננטות:** רשימת רשימות־עבר מקובצת לפי תאריך (כרטיס לכל רשימה עם תאריך, מספר פריטים, מי סיים), תצוגת פריטי רשימה היסטורית, כפתור "הוסף שוב לרשימה הנוכחית".
- **מצבים:** רשימת היסטוריה; תצוגת רשימה בודדת; תוצאות חיפוש.
- **מצב ריק:** "עדיין אין היסטוריה. רשימות שתסיימו יופיעו כאן".
- **מצב טעינה:** skeleton של כרטיסי היסטוריה.
- **מצב שגיאה:** "טעינת ההיסטוריה נכשלה" + retry.

## 7. ספריית קומפוננטות

### 7.1 כפתורים (Buttons)

| וריאנט | שימוש | סגנון |
|---|---|---|
| Primary | פעולה ראשית (אחת למסך) | רקע `--color-primary`, טקסט לבן, רדיוס 12px, גובה 48px, משקל 600 |
| Secondary | פעולה משנית | רקע surface, גבול `--color-border`, טקסט primary |
| Tonal | פעולה משנית רכה | רקע `--color-primary-soft`, טקסט primary |
| Ghost / Text | פעולות נמוכות עדיפות | ללא רקע, טקסט primary, hover רקע עדין |
| Destructive | מחיקה/עזיבה | טקסט/רקע `--color-error` |
| FAB | הוספת פריט | עיגול 56px, רקע primary, אייקון "+" לבן, `shadow-raised` |
| Icon Button | פעולות אייקון | 44×44px, רקע שקוף, hover עדין |

**מצבים לכל כפתור:** default, hover, pressed/active, focus (ring 2px primary), disabled (opacity 0.4), loading (ספינר במקום הטקסט).

### 7.2 שדות קלט (Inputs)

- **Text Input:** גובה 48px, רדיוס 12px, רקע `--color-surface-sunken`, גבול 1px על focus (`--color-primary`), label מעל, helper/error מתחת.
- **Segmented Code Input:** תאים נפרדים לקוד הזמנה (6 תווים), monospace, autofocus, מעבר אוטומטי בין תאים.
- **Search Input:** עם אייקון חיפוש מוביל, כפתור ניקוי, רקע sunken.
- **Quantity Stepper:** כפתורי −/+ משני צדי מספר.
- **מצבים:** default, focus, filled, error (גבול + טקסט אדום), disabled.

### 7.3 כרטיסים (Cards)

- **Base Card:** רקע surface, רדיוס 16px, padding `space-4`, `shadow-card`.
- **מצבים:** default, hover (דסקטופ — `shadow-raised`), pressed, selected (גבול primary 2px).

### 7.4 דיאלוגים / גיליונות (Dialogs / Sheets)

- **Bottom Sheet (מובייל):** עולה מלמטה, רדיוס עליון 24px, drag handle, overlay כהה (40% opacity), `shadow-modal`. ניתן לסגור בגרירה למטה או הקשה על overlay.
- **Centered Dialog (טאבלט/דסקטופ):** ממורכז, רוחב מקס' 480px, רדיוס 16px, overlay מעומעם.
- **Confirmation Dialog:** כותרת, טקסט, שני כפתורים (ביטול / אישור). פעולות הרסניות → כפתור אישור Destructive.

### 7.5 תפריטים נפתחים (Dropdowns)

- מופעלים מ-icon button או trigger; רקע surface, רדיוס 12px, `shadow-raised`, פריטים בגובה 44px, מפריד עדין, פריט הרסני באדום בתחתית.
- **מצבים:** closed, open (אנימציית scale/fade עדינה), פריט hover/focus.

### 7.6 הודעות צפות (Toasts)

- מופיעות בתחתית (מעל tab bar) במובייל, בפינה בדסקטופ.
- וריאנטים: success (ירוק), error (אדום), info (נייטרלי), undo (עם כפתור "ביטול").
- אורך חיים 3–4 שניות, ניתן לסגירה, `shadow-toast`.
- **שימוש קריטי:** סימון "נקנה"/מחיקת פריט מלווה ב-toast עם "ביטול" (Undo).

### 7.7 ניווט (Navigation)

- **Bottom Tab Bar (מובייל):** 3 טאבים (רשימה / היסטוריה / הגדרות), אייקון + תווית, טאב פעיל בצבע primary, גובה 56px + safe-area.
- **Side Nav (דסקטופ):** סרגל ימני קבוע 256px, פריטי ניווט אנכיים, כותרת משק בית למעלה, פרופיל למטה.
- **Top App Bar:** כותרת מסך, פעולת back/menu, פעולות הקשר מימין.

### 7.8 כרטיסי משק בית (Household Cards)

- **Member Card:** אווטר עגול (או ראשי תיבות על רקע צבעוני), שם, תפקיד (יוצר/חבר — badge), פעולת "הסר" (ל-יוצר בלבד, מופיעה ב-hover/swipe).
- **Invite Code Card:** קוד גדול monospace, כפתורי "העתק" + "שתף", הסבר קצר.

### 7.9 כרטיסי מוצר (Product Cards)

- **Item Row:** checkbox עגול (מצב לא־נקנה/נקנה), שם פריט (Body Large), כמות+יחידה (Footnote), אווטר זעיר של מי שהוסיף, אייקון/צבע קטגוריה מוביל.
- **מצבים:** active (לא נקנה), checked (קו חוצה + opacity מופחת + מעבר לתחתית), pending (סנכרון — אופסיטי + ספינר זעיר), swiping (חושף פעולות).
- **קיבוץ:** כותרות קבוצה לפי קטגוריה עם אייקון וצבע.

## 8. UX של רשימת הקניות

### 8.1 מצבי פריט (Item States)

1. **Active** — לא נקנה. checkbox ריק, טקסט מלא.
2. **Checked** — נקנה. וי בתוך checkbox ירוק, קו חוצה על הטקסט, opacity 0.5, הפריט נודד אוטומטית לתחתית הקבוצה / לאזור "נקנו".
3. **Pending** — בסנכרון (אופטימיסטי). מוצג מיד, opacity מופחת, אינדיקטור זעיר; הופך ל-Active כשהשמירה אושרה.
4. **Editing** — בעריכה inline או במודל.

### 8.2 תהליך השלמה (Completion Flow)

- הקשה על checkbox → אנימציה (וי + haptic) → toast עם "ביטול" (3 שניות) → הפריט נודד למטה.
- כשכל הפריטים סומנו → מופיע CTA "סיימתם לקנות? ארכבו את הרשימה" → ארכוב מעביר את הרשימה ל-`/history` ופותח רשימה נקייה.
- ניתן גם **swipe** ימינה לסימון מהיר, **swipe** שמאלה לחשיפת מחיקה.

### 8.3 מיון (Sorting)

אפשרויות מיון: **לפי קטגוריה** (ברירת מחדל — מתאים למסלול בסופר), **לפי סדר הוספה**, **א'-ב'**. הבחירה נשמרת לפי משתמש. פריטים שנקנו תמיד יורדים לתחתית הקבוצה שלהם.

### 8.4 קיבוץ (Grouping)

- קיבוץ ברירת מחדל לפי **קטגוריית מוצר** (פירות וירקות, מחלבה, מאפים, ניקיון...).
- כל קבוצה עם כותרת, אייקון וצבע אקסנט, וניתנת לקיפול (collapse).
- קטגוריה מוקצית אוטומטית לפי שם הפריט (ברירת מחדל "כללי"), ניתנת לשינוי ידני.

### 8.5 מצב ריק (Empty State)

- איור ידידותי (placeholder), כותרת "הרשימה ריקה", משפט מזמין, כפתור Primary גדול "הוסיפו פריט", וקישור משני "הזמינו את המשפחה".

## 9. UX של משק הבית

### 9.1 הצגת קוד הזמנה (Invite Code Presentation)

- הקוד מוצג גדול ובולט ב-monospace בתוך כרטיס ייעודי.
- כפתור ראשי "שתף" (Web Share API → וואטסאפ/הודעות) המשתף קישור `/join/:code`.
- כפתור משני "העתק" עם משוב מיידי ("הועתק ✓").
- מתחת: הסבר קצר "שלחו את הקוד לבני המשפחה כדי שיצטרפו".

### 9.2 הצגת רשימת חברים (Member List Presentation)

- רשימת Member Cards: אווטר/ראשי תיבות, שם, badge תפקיד ("יוצר"/"חבר").
- היוצר תמיד ראשון ומסומן.
- מונה חברים בכותרת ("4 חברים").

### 9.3 פעולות יוצר (Creator Actions)

- עריכת שם/אייקון משק הבית.
- הסרת חבר (אישור בדיאלוג Destructive).
- חידוש/ביטול קוד הזמנה.
- מחיקת משק בית (אישור כפול — פעולה הרסנית).
- חבר רגיל אינו רואה פעולות אלו.

### 9.4 תהליך עזיבת משק בית (Leave Household Flow)

- כפתור "עזיבת משק בית" באזור פעולות מסוכנות (אדום, תחתית הגדרות).
- דיאלוג אישור: "לעזוב את [שם משק הבית]? לא תהיה לך עוד גישה לרשימה המשותפת."
- **מקרה קצה — יוצר יחיד:** אם היוצר הוא החבר היחיד, "עזיבה" = "מחיקת משק הבית" (הסבר בדיאלוג).
- **מקרה קצה — יוצר עם חברים:** העזיבה מחייבת העברת בעלות לחבר אחר, או מחיקה. הדיאלוג מציע לבחור יוצר חדש.
- לאחר עזיבה → הפניה ל-`/onboarding`.

## 10. נגישות

### 10.1 ניגודיות (Contrast)

- כל הטקסט עומד ב-WCAG AA: 4.5:1 (רגיל), 3:1 (גדול/Bold 18px+).
- צבע אינו האינדיקטור היחיד — מצב "נקנה" משולב וי + קו חוצה + opacity, לא רק צבע.
- מצב Dark נבדק בנפרד לניגודיות.

### 10.2 ניווט מקלדת (Keyboard Navigation)

- כל פעולה נגישה במקלדת: Tab/Shift+Tab למעבר, Enter/Space להפעלה, Esc לסגירת מודלים.
- סדר Tab לוגי ותואם RTL.
- קיצורים בדסקטופ: `N` = פריט חדש, `/` = חיפוש, `Esc` = סגירה.
- מלכודת focus בתוך מודלים (focus trap).

### 10.3 מצבי Focus (Focus States)

- כל אלמנט אינטראקטיבי מקבל focus ring נראה: 2px `--color-primary` עם offset 2px.
- ה-focus לעולם לא מוסתר (`:focus-visible`).
- בפתיחת מודל — focus עובר לאלמנט הראשון; בסגירה — חוזר ל-trigger.

### 10.4 קוראי מסך (Screen Readers)

- HTML סמנטי (`<nav>`, `<main>`, `<button>`, `<ul>/<li>` לרשימות).
- `aria-label` לכל icon button; `aria-live` לאזורי סטטוס (toasts, סנכרון, מונה פריטים).
- checkbox של פריט עם `role`/`aria-checked` תקין ותווית = שם הפריט.
- מודלים עם `role="dialog"` + `aria-modal` + `aria-labelledby`.
- מצבי טעינה מוכרזים ("טוען רשימה").
- תמיכה מלאה ב-RTL (`dir="rtl"`, `lang="he"`).

## 12. הכנה ל-Cloud Design

סעיף ייעודי המתאר במדויק כיצד על Cloud Design לייצר את המסכים הוויזואליים הסופיים.

### 12.1 הוראות כלליות לכלל המסכים

- **שפה וכיוון:** עברית, RTL מלא (`dir="rtl"`, `lang="he"`). כל הפריסה, האייקונים הכיווניים והאנימציות תואמי RTL.
- **גופן:** Heebo (Google Fonts) כראשי, עם נפילה ל-system-ui. כותרות במשקל 700, גוף 400, הדגשות 600.
- **פלטה:** השתמשו אך ורק בטוקני הצבע מסעיף 3.1. Primary ירוק `#1FA463`, רקע לבן חם `#FBFAF8`, אקסנט כתום `#FF8A3D`. ספקו גרסת Light (ברירת מחדל) וגרסת Dark.
- **מרווחים:** סקאלת 4px בלבד (סעיף 3.3). padding מסך = 16px במובייל, 24px בטאבלט/דסקטופ.
- **רדיוס:** כרטיסים 16px, כפתורים/שדות 12px, גיליונות 24px עליון.
- **הצללה:** עדינה בלבד (סעיף 3.5). לעולם לא צללים כבדים/כהים.
- **טון ויזואלי:** 70% חמים וצבעוני וידידותי (Bring/AnyList) — אייקוני קטגוריה צבעוניים, כרטיסים מזמינים; 30% נקי ומדויק (Apple) — מרווח נדיב, טיפוגרפיה היררכית, אנימציות עדינות.

### 12.2 מפרט ייצור לכל מסך

עבור כל מסך, Cloud Design ייצר:
1. תצוגת **Mobile (375px)** — חובה.
2. תצוגת **Desktop (1024px+)** — למסכי הליבה (Dashboard, Workspace, History, Settings).
3. **מצב ברירת מחדל** + לפחות **מצב ריק אחד** או **מצב נוסף** רלוונטי (למשל: דשבורד מלא + דשבורד ריק; רשימה רגילה + רשימה שכולה נקנתה).
4. גרסת **Light** ו-**Dark** למסכי הליבה.

### 12.3 סדר עדיפויות לייצור

1. **Shopping Dashboard** (`/`) — מסך הליבה, החשוב ביותר. כולל מצב מלא + מצב ריק.
2. **Add Item Modal** — הפעולה הקריטית. כ-bottom sheet מעל הדשבורד.
3. **Shopping List / Workspace** — תצוגת עבודה עם קיבוץ ומיון.
4. **Onboarding + Create + Join** — מסע המשתמש החדש.
5. **Household Settings** — ניהול, כולל הצגת קוד וחברים.
6. **History** — ארכיון.
7. **Login + Auth Callback** — נקודות כניסה.

### 12.4 דגשים ויזואליים קריטיים ל-Cloud Design

- **מצב "נקנה"** של פריט חייב להיות מובחן בעליל: וי ירוק במעגל + קו חוצה + opacity 0.5 + ירידה לאזור "נקנו". אל תסתמכו על צבע בלבד.
- **אייקוני קטגוריה** צבעוניים (פירות/ירקות ירוק, מחלבה תכלת, מאפים חום-כתום, ניקיון סגול) — נותנים לרשימה את החמימות של Bring.
- **FAB** ה"+" הירוק חייב להופיע בכל מסך רשימה, פינה תחתונה, עם צל מורם.
- **אווטרים** של חברי משק הבית בכותרת ובפריטים — מחזקים את תחושת השיתוף.
- **Bottom Tab Bar** ב-3 טאבים במובייל; **Side Nav** קבוע בדסקטופ.
- **קוד הזמנה** ב-monospace גדול ובולט בכרטיס ייעודי עם כפתורי שתף/העתק.
- **מצבים ריקים** מזמינים, עם איור (placeholder) + CTA ברור — לא מסך ריק קר.

### 12.5 מה Cloud Design לא צריך לעשות

- לא ליצור קוד מימוש, חיבור ל-Supabase, או לוגיקת backend.
- לא להמציא מסכים שאינם במפרט זה.
- לא לחרוג מפלטת הצבעים וסקאלת המרווחים.
- לא להוסיף תוכן מילולי (filler) מעבר לדוגמאות הנדרשות להמחשה.

### 12.6 קריטריון הצלחה

המסכים שייוצרו נחשבים תקינים אם: (א) הם תואמים את ה-Design System בסעיף 3 במדויק; (ב) הם מכסים את כל המצבים שהוגדרו בסעיף 6; (ג) הם נקראים בבירור ב-RTL עברית; (ד) הם עומדים בדרישות הנגישות בסעיף 10; (ה) הם משקפים את תמהיל ה-70/30 (Bring/Apple) בטון הוויזואלי.
